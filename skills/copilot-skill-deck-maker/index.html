By surface

Where it works, and how.

Copilot CLI

The ask-Copilot install above works as-is. Skills in ~/.copilot/skills/ load when a session starts. Prefer to do it by hand?

New-Item -ItemType Directory -Force "$HOME\.copilot\skills\copilot-skill-deck-maker"
Copy-Item ".\SKILL.md" "$HOME\.copilot\skills\copilot-skill-deck-maker\SKILL.md" -Force

Clawpilot · desktop app

Same Copilot engine, so the skill runs. It does not auto-read ~/.copilot/skills. Save SKILL.md into any folder, then in Clawpilot open Settings → Skills, add that folder, and start a new session.

VS Code Copilot Chat

VS Code does not load CLI skills. Paste the contents of SKILL.md into .github/copilot-instructions.md (or .github/instructions/deck-maker.instructions.md), then start a new chat and ask Copilot to follow it.

Copilot Cowork · Microsoft 365

Cowork doesn't load filesystem skills. It's a different extensibility model. The method still works: paste the body of SKILL.md into your first message as instructions, then give it your notes. You just won't have it installed permanently.
