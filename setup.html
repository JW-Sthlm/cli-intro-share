Step 0

Get your GitHub account ready

Copilot CLI runs on GitHub. To use it, you need a GitHub account with Copilot access. As a Microsoft employee, this is provided for you — but you may need to set it up.

Do you have a GitHub account?

If you're not sure, go to github.com/login and try signing in. If you can't, you need to create one.

📝 No GitHub account yet? Create a free personal account at github.com/signup. Use any email you like — this is your personal account, not a Microsoft-managed one.

You might end up with two GitHub accounts

This is normal at Microsoft. Here's the difference:

AccountWhat it isHow to tell
PersonalYour own GitHub account — you created it yourselfUsername is whatever you chose (e.g., johansen42)
EMUA corporate account Microsoft created for youUsername ends in _microsoft (e.g., johansen_microsoft)

You might have both, just one, or neither yet. That's fine.

Which account for this session?

We recommend your personal account. It's simpler to use and works with everything we'll do in the session.

Connect your account to the Microsoft Copilot entitlement

Microsoft pays for your Copilot access — you just need to connect your account to that entitlement. This is the most important step.

👉 Action: Go to copilot.github.microsoft.com. Sign in, and follow the on-screen steps to verify your account is connected to the Microsoft Copilot license. If you see a green checkmark or confirmation — you're good.
⚠️ Can't access that portal? Search "GitHub Copilot QuickStart" on the R&I SharePoint site — it has step-by-step instructions and an automated verification tool. Or ask your manager or a colleague who already uses Copilot CLI.
Step 1

Install Copilot CLI

First: open PowerShell

PowerShell is a text-based tool for talking to your computer — like File Explorer, but you type commands instead of clicking. It comes pre-installed on Windows.

Press the Windows key, type PowerShell, and open the one that says "PowerShell 7" or "pwsh". Avoid the one called "Windows PowerShell" — it's an older version.

⚠️ Don't see PowerShell 7? You might only have the old version. Open "Windows PowerShell" instead — it will work for the install command. We'll sort out the version later.

Then: install Copilot CLI

Copy-paste this command into PowerShell and press Enter:

winget install GitHub.Copilot

Wait for it to finish (takes about a minute). Then close and reopen PowerShell — this is important, the new command won't be available until you do.

Verify it worked

copilot --version
✅ Pass: You see a version number (e.g., v1.0.x).
⚠️ winget not found? Open the Microsoft Store app, search for "App Installer", and install it. Then restart PowerShell and try again.
⚠️ copilot not found after install? Make sure you closed and reopened PowerShell. If still missing, restart your computer.
Step 2

Log in and test

Launch Copilot CLI:

copilot

You'll see an animated banner. Type:

/login

A browser window opens — sign in with your personal GitHub account.

Once logged in, test with a simple prompt:

What day is it today?
✅ Pass: You get an answer back. Type /exit to leave for now.
⚠️ Login fails or times out? Make sure you're on the corporate network or VPN. Verify your Copilot subscription at github.com/settings/copilot.
Step 3

Install Azure CLI (for PMX)

The PMX MCP server needs an Azure login to talk to Dynamics 365.

Install (if you don't have it)

winget install Microsoft.AzureCLI

Close and reopen PowerShell after installing.

Log in to the Microsoft tenant

az login --tenant 72f988bf-86f1-41af-91ab-2d7cd011db47

A browser window opens. Sign in with your Microsoft account. Done when it says "You have logged in."

✅ Pass: Run az account show --query "{name:name, user:user.name}" -o table — you see your account name.
Step 4

Configure MCP servers

MCP servers are connectors that let Copilot CLI talk to external tools. We're setting up three:

  1. PMX — partner management data in Dynamics 365
  2. GitHub — repos, issues, pull requests
  3. M365 — email, calendar, Teams

Option A: Config file (easiest)

Your session facilitator will share a pre-made config file. Save it to:

C:\Users\<your-username>\.copilot\mcp-config.json

Replace <your-username> with your actual Windows username.

Option B: Use the /mcp command

Inside Copilot CLI, type:

/mcp

This opens an interactive menu for adding MCP servers. Your facilitator will walk through this if needed.

💡 Tip: The config file approach is faster and less error-prone. Ask your facilitator for the file.
Step 5

Verify everything

Run through these five checks. All should pass before the session.

  • 1
    Copilot CLI is installed copilot --version
    ✅ You see a version number
  • 2
    You can log in and chat copilot → "Hello, what model are you using?"
    ✅ You get a response from the AI
  • 3
    Azure CLI is logged in az account show --query "{name:name, user:user.name}" -o table
    ✅ Shows your Microsoft account name
  • 4
    MCP servers are configured copilot → /env
    ✅ PMX, GitHub, and M365 listed under MCP servers
  • 5
    PMX data is accessible copilot → "Show me my PMX projects"
    ✅ You see project names from D365
⚠️ Something not working? Don't stress. We'll have 5 minutes at the start of the session to troubleshoot. But the more you can get done now, the more hands-on time you'll have.
